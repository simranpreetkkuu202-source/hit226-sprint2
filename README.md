# Personalised Study Focus Dashboard

This project is a ReactJS web application that demonstrates site personalisation through a simple study dashboard. The purpose of the site is to show how one interface can adjust its message, support style, and suggested actions for different users based on their selected profile and goal. The design focuses on clarity, ease of use, and quick interaction so users can move from broad intention to a more useful next step without feeling overloaded.

## Project report

### Purpose of site personalisation
The purpose of personalisation in this site is to make the interface feel more relevant to the person using it. Instead of showing the same message and support style to everyone, the dashboard changes the tone, guidance, and suggested plan according to the chosen user type and goal. This helps the interface feel more useful, more focused, and easier to understand.

### Likely users and persona
The likely users are students, busy working users, and beginner coders who need simple support when planning study or task based work.

A sample persona is a first year web technologies student who feels overloaded when instructions are broad. This user wants a clean layout, a short list of next steps, and content that feels encouraging rather than dense.

### User goals, assumption, and hypothesis
The user’s goal is to quickly identify what matters most and start working with less confusion.

Assumption: users engage better with a dashboard when the content reflects their current needs and confidence level.

Hypothesis: if the page personalises the language, priority guidance, and support style for the selected user type, then users will feel less overloaded and will identify their next action faster.

### How the example works for the user
The user enters a name, selects a user type, and chooses a main goal. The React app then updates the interface in real time. It shows a personalised message, a matching persona summary, and a suggested design response. The page also includes a button that reveals the design assumption and hypothesis, which demonstrates interactive behaviour in the web app.
